## 附录：AI 生成可行性分析

> 我们不使用专业动画软件（Aseprite/Spine），计划通过 AI 逐帧生成动画帧，再手动拼接为 Sprite Sheet。

### 核心挑战

AI 生成像素动画的最大问题不是"画不好"，而是**帧与帧之间不一致**——同一角色的两帧可能颜色、比例、细节都有偏差，导致播放时出现"抖动/闪烁"。

### 推荐工具与用途

| 工具 | 最适合 | 不适合 |
|------|--------|--------|
| **Midjourney / DALL·E** | 概念图、半身像立绘、场景概念、CG插画 | 像素Sprite、帧动画 |
| **Stable Diffusion + 像素LoRA** | 静态像素角色、道具图标、单帧Sprite | 多帧一致性 |
| **PixelLab / Pixelicious** | 将大图转像素风格、快速生成道具图标 | 角色动画 |
| **手动拼接（Aseprite Lite / Piskel）** | 帧序列排版、调色板统一、导出Sprite Sheet | — |

### 各类资产 AI 可行性评级

#### ★★★ 非常适合 AI（质量高、一致性好）

| 资产类型 | 方法 | 说明 |
|----------|------|------|
| **半身像立绘** | MJ/SD 生成 → 手动裁切 | 每个 NPC 只需1张，不需要帧间一致性 |
| **场景/背景** | MJ 生成概念 → Pixelicious 转像素 | 场景是静态的，没有帧间问题 |
| **道具/材料图标（25个）** | SD 逐个生成 32×32/48×48 小图 | 单个小图标，无需帧间一致 |
| **装备图标（~215个）** | SD 批量生成 + 稀有度边框叠加 | 可建立模板，替换内容 |
| **职业图标（28个）** | SD + 固定构图prompt | 独立图标，只需风格统一 |
| **UI 图标（~49个）** | SD / 手动绘制 | 简单几何图形，手动也很高效 |
| **CG插画/闪回（8段）** | MJ 逐段生成 | 每段是独立的画面，不要求帧间一致 |

#### ★★☆ 可以 AI 但需后处理

| 资产类型 | 方法 | 后处理 |
|----------|------|--------|
| **技能特效 VFX（~113个）** | SD 生成单帧 → 用参数微调生成多帧 | 手动筛选帧、统一调色板、去闪烁 |
| **怪物精灵（~55种 × ~4帧）** | SD 逐帧生成 → 固定seed微调 | 每帧检查一致性，可能需要手动修正 |
| **掉落光柱（6级）** | SD 生成基础纹理 → 代码缩放/旋转 | 光柱类特效对一致性要求低，容错高 |
| **UI 面板背景** | SD 生成纹理 → 九宫格拉伸 | 面板是固定UI，一张图即可 |

#### ★☆☆ 较难但可行（需大量后处理）

| 资产类型 | 方法 | 难点 |
|----------|------|------|
| **角色精灵（~168个 × ~6帧）** | SD 用同一 seed + frame_index 参数逐帧生成 | 每帧比例/颜色/细节漂移，需逐帧手动修正 |
| **Boss 精灵（~18个 × ~8帧）** | 同上 | Boss 体型大，帧间不一致更明显 |
| **武器模型（21个）** | SD 生成静态 → 使用时代码旋转 | 武器可做成静态图，旋转用代码实现，减少帧需求 |

#### ✗ 不推荐 AI

| 资产类型 | 原因 | 替代方案 |
|----------|------|----------|
| **角色连续动作动画** | 帧间一致性极难保证 | 改用代码驱动（缩放/旋转/位移/透明度变化）模拟动画 |
| **Boss 变身/多阶段** | 形态差异太大，AI 无法保持"同一个角色的不同形态" | 每个形态独立生成，用特效过渡掩盖切换 |

### 推荐的 AI 生成工作流

#### 工作流 A：立绘与场景（一套流程）
```
1. MJ 生成概念图
   prompt: "pixel art character portrait, [角色描述], 
           16-bit SNES style, dark fantasy, clean edges, 
           limited palette, 128x128 --ar 1:1"

2. Pixelicious 转换为像素
   上传 MJ 图 → 选择像素尺寸 → 导出

3. 手动微调
   Aseprite 打开 → 修正边缘 → 统一调色板 → 导出PNG
```

#### 工作流 B：角色/怪物静态帧（核心流程）
```
1. 确定角色设计稿（先用 MJ 出概念图确认方向）

2. SD 生成 Idle 帧（基准帧）
   使用像素LoRA + ControlNet(pose) + 固定seed=42
   prompt: "[角色名], pixel art sprite, [职业/种族], 
           [配色], [装备描述], game asset, spritesheet style, 
           64x64, limited 16 colors, clean pixel edges"

3. 保持 seed 不变，调整 prompt 生成其余帧
   每帧在 prompt 末尾加 "[动作描述] frame 2/6"
   但需手动筛选：比例/颜色/位置一致的帧才保留

4. 手动调色板统一
   Aseprite → 导入所有帧 → "颜色表统一" → 逐帧修正偏移
```

#### 工作流 C：图标/道具（批量高效）
```
1. 用脚本批量生成 SD 请求
   对每个材料名拼 prompt 模板

2. prompt: "pixel art game icon, [物品名], 
           32x32, isolated on transparent, 
           RPG inventory style, clean edges, limited palette"

3. 批量导出 + 按稀有度叠加边框
   代码给白/绿/蓝/紫/金/红色边框
```

#### 工作流 D：技能特效 VFX（帧序列专用）
```
1. 先用 SD 生成一张满意的关键帧（特效最亮的瞬间）

2. 用 img2img + 低 denoising strength(0.3-0.5) 
   逐步生成前后帧
   
   或用 Deforum 插件生成连续过渡帧

3. 手动选6-8帧 → 统一透明背景 → 拼成 spritesheet

4. 可选：代码层面做缩放/旋转/Tween 减少帧数需求
   例如：6帧AI生成 + 代码插值 = 流畅动画
```

### 实战建议

1. **先做到"能跑"，再做到"好看"** — 第一批用简单几何图形+纯色做占位符，系统通了再替换

2. **复用胜过新造** — 同职业系共享基础动画帧（例如所有战士 Idle 帧相同，只换武器颜色），把 168 个精灵的实际工作量压到约 30 套动画

3. **特效用代码补** — 刀光、冲击波、粒子这类线形/圆形特效，用 Canvas API 画比 AI 生成更快更稳

4. **建立统一调色板** — 所有角色共用一套 32 色像素调色板，能大幅减少 AI 生成的颜色漂移问题

### 按可行性重排优先级

| 优先级 | 资产 | 方法 | 预计人天 |
|--------|------|------|----------|
| P0 立即开始 | 28 职业图标 + 25 材料图标 | 工作流C 批量 | 2天 |

| P1 | 8 段剧情 CG/闪回 | 工作流A | 2天 |
