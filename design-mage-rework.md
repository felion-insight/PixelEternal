```
法师 (Mage) — 远程魔法，元素掌控
```

## 2. 核心问题分析

### 问题一：贤者/先知 — 定位严重混乱

**症状**：

- 贤者设计文档称"治疗+辅助"，但热键栏**保留火球术**（纯攻击技能）
- 一转主题是"神圣治疗"，二转突然变成"时间操控"——主题完全断裂
- 玩家困惑：我是治疗？是输出？还是半吊子两者？

**根因**：技能进化路线缺乏规划。贤者的 `core1` 没有进化自己的治疗技能，而是保留了火球；到了先知又跳到时空主题。

---

### 问题二：三个分支共用法力 — 资源零差异化

**症状**：

- 巫师、贤者、术士都使用法力（或名义上的 soul_shard）
- 术士的 soul_shard 在底层被映射到 mana family，实际就是法力
- 三个分支的"节奏感"完全一致，唯一的区别是技能颜色

**对比**：

```
弓箭手系：
  游侠   → pet_energy   → 宠物攻击/指令产资源
  神射手 → ammo         → 装填机制，资源阶段性断档
  风行者 → wind_mark    → 移动/暴击产资源
→ 三种完全不同的资源节奏！

法师系：
  巫师 → mana → 自动回复
  贤者 → mana → 自动回复  
  术士 → mana → 自动回复
→ 一样。
```

---

### 问题三：术士的"DOT+召唤"形同虚设

**症状**：

- 文档称"DOT+召唤DPS"，实际只有一个召唤技能（召唤骷髅）
- 暗影弹只是火球换颜色，没有 DOT 机制
- 无诅咒体系、无 DOT 叠加/扩散/引爆、无灵魂碎片获取→消耗的闭环

**实际技能分析**：

```
术士热键：
  暗影弹     → 火球换皮，无 DOT
  冰霜新星   → 基础技能保留，与暗影主题无关
  召唤骷髅   → 唯一符合定位的技能
  暗黑祭品   → 消耗生命召唤，但与 DOT 无关
```

---

### 问题四：元素交互有名无实

**症状**：

- `status-synergy-config.json` 中有 `thermal_shock` (烧+冻=真伤)、`conductive_freeze` (感电+冻=增伤) 等协同规则
- 但这些是被动的条件判断，不是专属机制
- 元素协奏的描述是"根据目标已有状态触发额外效果"——本质就是 `if(burn) doFire(); if(frostbite) doIce()`
- 没有任何职业专有技能/被动利用这套协同

**真正的问题**：元素交互应该是巫师的**核心玩法**，而不只是全局规则。

---

### 问题五：零职业被动，零专属系统文件

**症状**：

- 项目中 `warrior-passive.js`、`ranger-pet-system.js`、`beastmaster-system.js`、`marksman-precision-system.js` 等系统文件齐全
- 法师系：**零个专属 JS 文件**
- 全局 class-config 中只有 berserker/destroyer 有 `classPassive`
- 没有任何法师分支拥有 `classPassive`

---

### 问题六：技能进化 = 换皮 + 数值膨胀

**症状**：

| 进化链 | 基础 | 一转 | 二转 |
|--------|------|------|------|
| core1 | 火球术 (2.0x, 半径70) | 烈焰弹 (2.8x, 半径105) | 熔岩风暴 (更高数值) |
| survival | 元素风暴 | 元素协奏 | 陨石 |

→ 本质只是 **damage × 1.4** + **radius × 1.5**，没有机制质变。

**对比战士系**：守护者的 `core2` 从 `guardian_charge`（冲锋）进化为 `sacred_bond`（团队生命链接）——**机制完全不同**。

## 3. 业界参考

### 3.1 法师/施法者职业设计趋势

| 游戏 | 法师类设计 | 核心机制亮点 |
|------|-----------|-------------|
| **FFXIV 黑魔** | 星极火/灵极冰相位切换 | 爆发期(火)↔回复期(冰)主动循环，位置规划是核心 |
| **FFXIV 占星** | 卡牌系统 + 治疗 | 随机抽牌为队友提供不同增益，决策压力大 |
| **Diablo 4 巫师** | 附魔槽位 | 将主动技能放入附魔槽获得被动效果，构建自由度高 |
| **Diablo 4 亡灵法师** | 亡者之书 | 三种仆从（战士/法师/魔像）分别定制，每类可选3种变体 |
| **Path of Exile** | 元素异常体系 | 冰缓/冰冻、感电、点燃 各有机制深度，非伤害异常(焦灼/脆骨/筋疲) |
| **Last Epoch 符文大师** | 符文召唤 | 按固定顺序施放技能组合出不同符文效果 |
| **World of Warcraft 戒律牧** | 救赎 | 伤害输出的百分比转化为队友治疗，统一输出和治疗 |
| **原神** | 元素反应 | 火+水=蒸发(1.5x/2x)、火+冰=融化(1.5x/2x)、雷+火=超载(AOE)，元素组合≠元素堆砌 |

### 3.2 关键设计原则提炼

1. **资源 = 节奏**：不同分支的资源获取/消耗方式应完全不同，让手感产生质的差异
2. **机制 > 数值**：转职应改变"你怎么玩"，而非"你打多少伤害"
3. **主题一致性**：一转到二转的视觉和机制主题应渐进强化，而非突变
4. **专属系统**：每个分支应有其他分支无法复制的核心机制

## 4. 重做方案总览

### 4.1 设计原则

1. **三支分化**：巫师(元素掌控)、贤者(时间治疗)、术士(DOT召唤) 机制完全独立
2. **资源独立**：三种专属资源，节奏各不相同
3. **机制质变**：转职时引入新机制，不是数值叠加
4. **渐进主题**：一转定基调，二转强化至极致

### 4.2 重做后职业树

```
法师 (Mage) — 魔法学徒，三系入门
├── 巫师 (Wizard)       → 大魔导师 (Archmage)
│     元素相位循环        → 元素解放 + 四元素共鸣
│     🔥爆发 / ❄️回复     → 元素融合 + 冷却刷新
│
├── 贤者 (Sage)          → 先知 (Oracle)
│     时光治疗 + 预知       → 时间操控 + 命运改写
│     ⏳给队友"预借"生命    → 全队时间逆转
│
└── 术士 (Warlock)       → 死灵法师 (Necromancer)
     诅咒DOT + 灵魂管理    → 亡灵军团 + DOT爆发
     💀击杀产生灵魂碎片    → 灵魂→召唤/DOT引爆 闭环
```

### 4.3 资源系统对比

| 职业 | 资源 | 获取方式 | 消耗方式 | 节奏特征 |
|------|------|----------|----------|----------|
| **巫师** | 元素能量 | 同系连续施法积累 | 跨系施法消耗/触发共鸣 | 🔥爆发期猛攻 ↔ ❄️回复期蓄力 |
| **贤者** | 时光沙漏 | 成功预判伤害获得 | 释放时间法术 | ⏳预判成功=资源正循环，失败=资源亏损 |
| **术士** | 灵魂碎片 | 敌人死亡/DOT每跳 | 召唤/引爆DOT/强化诅咒 | 💀战场管理：何时消耗、何时囤积 |

## 5. 巫师 → 大魔导师 重做

### 5.1 核心机制：元素相位 (Elemental Phase)

#### 概念

巫师在使用不同元素技能后进入不同的**元素相位**，每个相位改变全局属性。玩家需要在两个相位之间主动切换，形成"爆发→蓄力→爆发"的节奏循环。

#### 相位机制

```
         🔥 灼热相位 (Fire Phase)
         ┌─────────────────────────┐
         │ 火系伤害     +25%        │
         │ 法力消耗     +50%        │
         │ 法力回复     -50%        │
         │ 移动速度     -10%        │
         │ → 站桩猛轰，但资源快速枯竭 │
         └───────────┬─────────────┘
                     │  使用冰系技能
                     ▼
         ❄️ 霜寒相位 (Frost Phase)
         ┌─────────────────────────┐
         │ 冰系伤害     +15%        │
         │ 法力消耗     -30%        │
         │ 法力回复     +200%       │
         │ 移动速度     +15%        │
         │ → 灵活走位，快速回蓝     │
         └───────────┬─────────────┘
                     │  使用火系技能
                     ▼
              🔥 回到灼热相位
```

**相位切换本身也有收益**：切换瞬间触发一次"元素脉冲"（半径80，无消耗，造成 1.0x 混合元素伤害 + 施加对应状态）。

#### 相位机制的玩法价值

- **新手友好**：蓝不够用 → 切冰回蓝 → 蓝满了 → 切火爆发的直觉循环
- **高手深度**：掌握切相位时机（boss无敌期切冰回蓝/攒冷却，输出窗口切火爆伤害最大化）
- **团队协调**：需要治疗保护时切冰增强生存；队伍爆发期切火协同输出

---

### 5.2 元素共鸣系统

当目标身上同时存在**灼烧**和**冻伤**（由不同玩家或连续施法造成），触发元素共鸣：

| 共鸣类型 | 条件组合 | 效果 | 视觉 |
|----------|----------|------|------|
| 🔥💥 **热冲击** | 灼烧 + 冻伤 | 立即结算全部灼烧剩余伤害 × 2.0 + 击退 | 蒸汽爆炸 |
| ❄️⚡ **超导** | 冻伤 + 感电 | 降防 30% × 5s + AoE扩散伤害 | 冰晶碎裂+电弧 |
| ⚡🔥 **过载** | 感电 + 灼烧 | 触发范围爆炸（半径120, 1.5x）+ 施加灼烧 | 雷火风暴 |
| 🌪️🌀 **元素解放** | 触发任意3次共鸣 | 进入解放状态：全伤+50%，冷却-60%，持续6s | 四色光环 |

## 6. 贤者 → 先知 重做

### 6.1 核心机制：时光沙漏 (Chronos Sand)

#### 概念

贤者是一个**预判型治疗**职业。其核心资源"时光沙漏"通过**成功预判并抵消伤害**来获得，用于释放更强力的时间法术。

#### 设计哲学

- **不是"掉血→加血"的被动治疗**，而是"预判伤害→提前防护→成功防护=资源正循环"
- 预判对了 → 获得大量资源 → 可以释放更强技能
- 预判错了 → 浪费资源 → 需要普攻积累
- 这构成了贤者的核心**技巧上限**

#### 资源机制

```
时光沙漏 (Chronos Sand)
├── 最大：100
├── 获取方式：
│   ├── 预知之盾成功吸收伤害：+25（核心来源）
│   ├── 时间弹命中敌人：+5
│   └── 自然回复：+3/s（极慢）
└── 消耗方式：
    ├── 释放时间法术
    └── 预知之盾施放：-30
```

---

### 6.2 技能重设计方案

#### 基础法师（不变）

#### 贤者（一转，20-39级）

**彻底移除火球术。** 贤者是治疗，不需要输出技能。

| 槽位 | 技能名 | 描述 |
|------|--------|------|
| basic | **时序弹** | 发射时间能量弹：命中敌人=1.0x伤害+产生5沙漏；命中队友=0.8x魔法攻击的治疗+产生3沙漏 |
| core1 | **预知之盾** | 为目标施加护盾（吸收量=2.0x魔攻），持续4s。若护盾在期间被消耗>50%，获得25沙漏+技能CD-3s。若护盾未被消耗，返还15沙漏。消耗30沙漏 |

## 7. 术士 → 死灵法师 重做

### 7.1 核心机制：灵魂碎片经济 (Soul Shard Economy)

#### 概念

灵魂碎片是术士的真正专属资源（不再映射到mana）。碎片通过**战场管理**获得，通过消耗来实现爆发。

#### 资源机制

```
灵魂碎片 (Soul Shard)
├── 最大：8（不能通过属性增加）
├── 获取方式：
│   ├── 每个敌人死亡：+1（杂兵战富裕）
│   ├── 痛苦诅咒每跳：8%概率 +1
│   └── 生命汲取完整引导：+2
└── 最大容量小（8），鼓励"消耗 → 补充"循环
```

#### 设计哲学

- **多目标战斗**（杂兵多）→ 碎片充裕 → 频繁召唤/引爆
- **少目标战斗**（Boss战）→ 碎片稀缺 → 需要精细管理DOT跳和生命汲取来获取

---

### 7.2 DOT体系

术士的DOT不再只是"附加状态"，而是其核心玩法：

```
痛苦诅咒 (Agony Curse)
├── 类型：可叠加 DOT
├── 最高层数：5层
├── 每层伤害：0.15x 魔攻/s
├── 持续时间：8s（刷新叠加时刷新持续时间）
├── 满层效果：每跳有 15% 概率产生 1 灵魂碎片
└── 交互：消耗所有层数触发"灵魂收割"时，每层造成 1.0x 魔攻

腐蚀 (Corruption)  
├── 类型：不可叠加 DOT（由特定技能施加）
├── 伤害：0.3x 魔攻/s
├── 持续时间：6s
├── 特殊效果：受腐蚀影响的敌人，骷髅对其伤害 +30%
└── 交互：被"生命汲取"引导期间，腐蚀伤害频率翻倍
```
