# 攻击卡肉效果实现方案（无代码版）

## 1. 概述

**卡肉（Hit Stop / Hit Freeze）** 是动作游戏中提升打击感的常用技术，指攻击命中瞬间，游戏逻辑短暂冻结，同时配合视觉特效，营造“重击”的顿挫感。  
本方案针对 **HTML5 Canvas + JavaScript** 环境，提供一套完整、可落地的卡肉效果设计方案，不包含具体代码，仅描述机制、流程与注意事项。

---

## 2. 核心机制

### 2.1 触发条件
- **近战武器**命中敌人时：触发完整卡肉效果（逻辑暂停 + 视觉特效）。
- **远程武器**（弓箭、法杖）命中时：仅触发轻量版（不暂停逻辑，仅震动与闪光）。
- 可扩展：暴击、背刺、破防等特殊命中可延长卡肉时长或增强特效。

### 2.2 逻辑暂停机制
- 游戏主循环由 `requestAnimationFrame` 驱动，每一帧执行 **更新（Update）** 和 **渲染（Draw）** 两个阶段。
- 卡肉期间：
  - **跳过更新阶段**：玩家输入、移动、碰撞、技能冷却、敌人 AI 等逻辑均不执行。
  - **渲染阶段正常执行**：屏幕震动、闪光、冲击线等特效依然绘制，形成“画面冻结但特效残留”的视觉。
- 暂停时长通常为 **50～100 毫秒**，应设计为可配置参数，便于调优。

### 2.3 计时器叠加规则
- 使用一个 `hitStopTimer` 变量（单位为毫秒或帧计数）。
- 命中时：`hitStopTimer = max(hitStopTimer, duration)`，不累加。
- 每帧更新时：若 `hitStopTimer > 0`，则减去经过时间，期间跳过逻辑更新。

## 3. 视觉反馈设计

### 3.1 屏幕震动
- 原理：在渲染阶段，对 Canvas 的 `translate` 变换施加随机偏移。
- 实现：
  - 震动幅度（振幅）通常 3～5 像素。
  - 卡肉触发时设置 `shakeIntensity = 5`，每帧衰减（如乘以 0.9），直至小于 0.1 时归零。
  - 偏移量 = `(Math.random() - 0.5) * shakeIntensity * 2`。
- 轻量版（远程）：振幅减半，衰减速度不变。

### 3.2 命中闪光
- 原理：在命中坐标处绘制一个径向渐变或半透明圆形，迅速放大并消失。
- 生命周期：持续约 0.1 秒（与卡肉时长同步）。
- 实现方式：
  - 使用粒子对象：`{ x, y, radius, alpha, life }`。
  - 每帧半径增加（如 +2px），透明度递减（如 alpha *= 0.8）。
  - 生命结束（alpha ≤ 0 或 radius ≥ 阈值）后移出数组。

### 3.3 冲击线（Impact Lines）
- 原理：从命中点向外辐射的直线段，表现冲击波扩散。
- 数量：4～8 条，方向均匀分布（每 45° 一条）。
- 实现：
  - 每条线对象：`{ startX, startY, angle, length, alpha }`。
  - 每帧长度增加（如 +3px），透明度递减。
  - 绘制时使用 `lineWidth = 2`，`strokeStyle = "rgba(255, 255, 255, alpha)"`。
- 可与闪光共用同一个粒子管理器。

## 4. 与游戏循环的集成方式

### 4.1 标准游戏循环结构（无代码说明）
- 每一帧执行以下顺序：
  1. 计算 `deltaTime`（时间差，限制最大值）。
  2. **更新阶段**：
     - 若 `hitStopTimer > 0`，则 `hitStopTimer -= deltaTime`，**跳过所有逻辑更新**。
     - 否则，正常更新玩家、敌人、子弹、碰撞等。
  3. **渲染阶段**：
     - 应用屏幕震动偏移（`ctx.translate`）。
     - 绘制所有游戏元素（背景、地图、玩家、敌人、特效）。
     - 叠加绘制命中闪光和冲击线（独立于游戏实体，始终显示）。
     - 恢复 Canvas 变换。
  4. 清理超出生命周期的特效粒子。

### 4.2 远程与近战分支
- 在命中检测函数中，根据 `isRanged` 标志决定：
  - `true`：仅触发轻量震动 + 闪光，不修改 `hitStopTimer`。
  - `false`：设置 `hitStopTimer = hitStopDuration`，并启动全特效。

## 5. 性能与优化建议

- **粒子数量限制**：闪光和冲击线最多同时存在 10 个实例，超出则丢弃旧粒子。
- **震动计算**：每帧重新生成随机偏移，无额外开销。
- **命中检测频率**：卡肉期间仍可进行命中检测（但通常无新攻击输入），避免重复触发导致计时器重置。建议在命中函数开头判断：若已处于卡肉状态且为新近战命中，则仅重置计时器，不重复创建特效。
- **移动端适配**：震动幅度不宜过大（≤3 像素），避免眩晕感。

## 6. 参数调优建议

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| 完整卡肉时长 | 70ms | 适中，兼顾顿挫感与流畅度 |
| 远程轻量震动 | 2px 振幅，0.07s 衰减 | 避免打断节奏 |
| 闪光初始半径 | 8px | 小于命中判定圈 |
| 闪光最大半径 | 24px | 随 alpha 归零而消失 |
| 冲击线数量 | 6 条 | 均匀分布 |
| 冲击线扩展速度 | 每帧 +4px | 约 0.1 秒延伸 24px |
| 震动衰减系数 | 0.85/帧（60fps 下） | 约 0.1 秒归零 |

## 7. 扩展与变体

- **根据武器重量调整时长**：匕首 40ms，大剑 90ms。
- **根据命中部位**：头部或弱点命中时长增加 20%。
- **连续卡肉保护**：若 0.5 秒内连续触发多次，后续卡肉时长衰减 30%，避免游戏过于粘滞。
- **与音效配合**：卡肉暂停期间不应暂停音频播放，但可短暂降低音高或播放特殊“顿挫”音效。

---

## 8. 测试验证方法

1. **单元测试**：模拟命中调用，检查 `hitStopTimer` 是否正确设置与衰减。
2. **视觉验收**：
   - 近战命中时画面明显停顿，伴随白光和冲击线。
   - 远程命中无停顿，仅有轻微震动。
3. **连续命中**：快速多次攻击，卡肉时间应为最后一次命中的时长，不累加。
4. **性能**：特效粒子数量不超过上限，无内存泄漏。

---

## 9. 总结

本方案提供了一套完整的攻击卡肉效果设计，包含逻辑暂停机制、屏幕震动、闪光与冲击线特效，以及远程/近战的差异化处理。无需依赖外部库，完全基于 HTML5 Canvas 与 JavaScript 的标准特性实现。开发者可根据上述步骤自行编码实现，并灵活调优参数以达到理想的打击感。
