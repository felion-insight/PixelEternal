您的深阶装备贴图相似度过高，主要是因为当前提示词中缺少对 **主题视觉特征** 和 **品质复杂度层级** 的明确区分。以下从提示词设计、生成流程、质量控制三个维度提出具体修改建议，确保不同主题、不同品质的装备在外观上具备明显辨识度。

## 一、为每个主题建立独特的视觉“指纹”

每个主题应有**专属的主色调、核心材质、装饰符号**，并在提示词中强制出现。

### 1. 主题视觉指纹表（建议）

| 主题 | 主色调 | 核心材质 | 装饰符号 | 特殊光效/纹理 |
|------|--------|----------|----------|----------------|
| 渊隙 | 暗紫、深灰 | 裂隙岩、暗金属 | 裂纹、发光缝隙 | 紫光裂缝、暗流涌动 |
| 虚印 | 银蓝、苍白 | 符文石、灵体 | 发光符文、幽灵轮廓 | 蓝白辉光、半透明材质 |
| 腐噬 | 暗绿、锈棕 | 腐化金属、黏液 | 菌斑、腐蚀孔洞 | 绿色毒雾、滴落黏液 |
| 黑曜 | 纯黑、暗红 | 黑曜石、火山岩 | 尖锐棱角、岩浆纹 | 暗红裂纹、镜面反光 |
| 终幕 | 深紫、暗金 | 帷幕布、古铜 | 布纹、幕帘褶皱 | 阴影渐变、暗金光晕 |

### 2. 提示词中强制引用示例
在生成每个主题装备时，必须将对应“主色调+材质+装饰符号”写入提示词：
```
渊隙主题头盔，暗紫色金属，表面布满发光裂缝，岩石质感，像素风...
```
这样不同主题的装备在颜色、材质、细节上立刻拉开差距。

## 二、品质字映射为明确的复杂度层级

目前“凡/良/湛/炽/曜”仅表示品质，但未在视觉上体现。应将其映射为从朴素到华丽的**复杂度阶梯**，并在提示词中使用具体描述。

### 建议的复杂度映射表

| 品质字 | 视觉描述（英文提示词用） | 示例修饰词（中文） |
|--------|--------------------------|-------------------|
| 凡 | `simple, minimal decoration, flat surfaces` | 简洁，无装饰，光面 |
| 良 | `some details, minor engravings, small rivets` | 少量纹路，铆钉，基础雕刻 |
| 湛 | `intricate engravings, gem inlay, layered plates` | 精密雕刻，镶嵌宝石，分层结构 |
| 炽 | `ornate, glowing accents, elaborate filigree, runic symbols` | 华丽，发光元素，繁复纹样，符文 |
| 曜 | `magnificent, radiant aura, legendary, particle effects, dynamic glow` | 辉煌，光效，粒子，传说感 |

### 提示词中使用方式
```
虚印·曜 近战武器 → 符文剑身，蓝白辉光，粒子环绕，华丽雕刻，传说感。
腐噬·凡 胸甲 → 腐化金属，绿色菌斑，简单无装饰，粗糙表面。
```

## 三、等级（主题）与品质的组合体现

不同主题对应不同等级（25~60级），但主题本身已暗示等级强度。建议在提示词中通过**材质老化程度、结构复杂度、光效强度**来间接体现等级差异。

- **低等级主题（渊隙、虚印）**：材质粗犷、裂缝简单、光效微弱。
- **中等级主题（腐噬、黑曜、终幕）**：材质腐蚀/锋利、细节丰富、光效明显。
- **高等级主题（星骸、裂点、终焉）**：材质稀有（陨铁、裂晶）、结构破碎/重构、光效强烈。

例如，同样“曜”品质：
- 渊隙·曜：暗紫金属，裂缝中涌出浓烈紫光，但材质仍有岩石感。
- 终焉·曜：暗红金属，裂纹中流淌岩浆，整体散发毁灭气息。

## 四、优化提示词模板（以武器为例）

将上述要素整合成结构化的提示词模板，供 GPT 规划时使用。

### 通用模板（可直接用于批量生成）
```
[主题视觉关键词] + [品质复杂度描述] + [部位特征] + 暗黑像素风，16-bit，纯黑背景，道具图标，透明底
```

**示例（渊隙·曜 近战武器）**
```
近战武器，渊隙主题：暗紫色金属，表面布满发光紫色裂缝，岩石质感，华丽雕刻，剑身镶嵌暗红宝石，辐射光效，粒子环绕，暗黑像素风，16-bit，纯黑背景，道具图标，透明底
```

**示例（终幕·凡 头盔）**
```
头盔，终幕主题：深紫色布纹与暗金色骨架，简单无装饰，帷幕褶皱造型，朴素表面，暗黑像素风，16-bit，纯黑背景，道具图标，透明底
```

## 五、在 `art_generator.py` 中的落地建议

### 1. 修改批量生成逻辑
当前 `run_batch_weapon_textures` 函数仅调用 GPT 规划一次，但未强制主题差异。可以：
- 在调用 `call_gpt_plan_weapon_texture` 前，将“主题视觉指纹表”和“品质复杂度表”作为系统提示或上下文注入，要求 GPT 在 `image_prompt` 中体现。
- 或者，完全放弃 GPT 规划，直接使用上述模板拼接（更快、更可控），通过循环生成所有组合。

### 2. 新增批量深阶装备生成函数
可仿照 `run_batch_weapon_textures` 增加一个 `run_batch_deep_equipment(theme, quality)`，内部按部位循环，直接使用模板生成提示词，调用 `generate_image` 并保存。这样可精确控制每个主题每个品质的视觉效果。

### 3. 后处理增强（可选）
生成图片后，可对特定主题或品质添加统一的滤镜、边缘光效或缩放，进一步提升区分度。但提示词是首要因素。

## 六、验证与迭代

1. **生成小样本测试**：先为每个主题生成“凡”和“曜”两个极端品质的武器，观察视觉差异是否明显。
2. **收集反馈**：将生成的图片混合，让团队成员尝试分类主题和品质，验证辨识度。
3. **微调提示词**：根据结果调整主题色彩、品质描述词的强度。
